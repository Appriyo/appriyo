# LAYER 1 — PURPOSE & STRATEGY

---

## 1.1 Hero Objective

> **The hero exists to do one thing: make a business owner with a manual workflow problem immediately recognize that Appriyo is built for them — and want to take the next step.**

It is NOT a brand statement.
It is NOT a technology showcase.
It is NOT a design portfolio piece.

It is the opening sentence of a conversation between Appriyo and a frustrated business owner.

---

## 1.2 The Three-Second Rule

A visitor landing on the Appriyo homepage has 3 seconds before they decide to stay or leave.

In those 3 seconds, the hero must answer three questions — in this exact order:

| Second | Question the visitor asks | What the hero must answer                         |
| ------ | ------------------------- | ------------------------------------------------- |
| 0–1s   | "What is this?"           | A company that replaces manual business processes |
| 1–2s   | "Is this for me?"         | Yes — if you're running a small business manually |
| 2–3s   | "What do I do next?"      | Click "Let's Talk About Your Business"            |

If the hero fails any of these three, the visitor bounces.

---

## 1.3 Hero Goal

**Primary:** Convert — drive the visitor toward contacting Appriyo.
**Secondary:** Position — establish Appriyo as the simple, practical alternative to generic IT agencies.
**NOT:** Educate in depth (that's the services and solutions sections below).

---

## 1.4 Success Criteria

The hero is successful when:

```
[ ] A non-technical 45-year-old repair shop owner reads it and thinks:
    "This is exactly what I need."

[ ] The visitor scrolls down OR clicks a CTA within 5 seconds.

[ ] The bounce rate on the homepage is below 60%.

[ ] When asked in a post-consultation call:
    "What made you reach out?" — the visitor references the problem statement from the hero.

[ ] The hero feels nothing like a generic SaaS landing page.

[ ] A developer, non-technical business owner, and team member all agree:
    "You understand what we're about in under 3 seconds."
```

---

## 1.5 What This Hero Is NOT Trying to Do

| Not the hero's job         | Why                                                   |
| -------------------------- | ----------------------------------------------------- |
| Explain all 4 services     | That's the Services section                           |
| List product features      | That's the Products section                           |
| Build full trust           | That's Why Appriyo + About sections                   |
| Impress with design        | Design serves the message — not the other way around  |
| Target technical audiences | The primary visitor is a non-technical business owner |

| -------------------- | ------------------------------------------------------------------------ |

| Structure            | Question OR bold statement — never a company tagline                     |
| Tone                 | Direct, conversational, calm — not dramatic                              |

## 2.7 CTA System

### Primary CTA

```
Let's Talk About Your Business
```

**Why this phrasing:**

- "Let's" — collaborative, not salesy
- "Talk" — low commitment (not "Buy", not "Request a Demo")
- "About Your Business" — confirms this is a personalized conversation, not a generic form

**Destination:** Scrolls to `#contact` section on the homepage

### Secondary CTA

```
See What We Build
```

**Why this phrasing:**

- Low-pressure — no commitment implied
- Directs to products — lets the visitor build confidence through proof before contacting
- Short — doesn't compete with the primary CTA

**Destination:** Scrolls to `#products` section on the homepage

### CTA Rules

| Rule                         | Detail                                                                                   |
| ---------------------------- | ---------------------------------------------------------------------------------------- |
| Primary CTA max length       | 5 words                                                                                  |
| Secondary CTA max length     | 4 words                                                                                  |
| Primary CTA style            | Filled button — primary blue background, white text                                      |
| Secondary CTA style          | Outline button — border only, no fill                                                    |
| Forbidden primary CTA text   | "Get Started", "Contact Us", "Submit", "Learn More", "Click Here"                        |
| Forbidden secondary CTA text | "Explore", "Discover", "View More"                                                       |
| CTA intent                   | Primary = start a conversation. Secondary = build confidence before contacting.          |
| CTA placement                | Primary on left, secondary on right (desktop). Primary on top, secondary below (mobile). |
| CTA spacing                  | Minimum 16px gap between primary and secondary                                           |
| Hover behavior               | Primary darkens. Secondary fills slightly. Both subtle.                                  |

## 3.1 Layout Decision

**Chosen layout: Centered, Single Column**

Rationale:

- Appriyo's primary visitor is a non-technical SMB owner — they need full focus on the message, not distraction from a visual element
- At this stage, no product UI mockup is ready that would justify a split layout
- Centered layout puts 100% visual weight on the message — which is where it belongs
- A split layout with a placeholder or stock illustration signals "generic" — exactly what Appriyo rejects

**When to revisit:** If a high-quality product UI mockup (Amar Repair dashboard) is available, reconsider a split layout in Phase 2.

# LAYER 4 — VISUAL DESIGN SYSTEM

## 4.1 Background

### Decision: Dark, subtle geometric pattern

**Exact specification:**

- Base color: `#0b0f19` (the site's `--color-bg` token)
- Overlay: An extremely low-opacity (3–5%) dot grid or fine line grid pattern
- Pattern color: `#2563eb` (primary blue) at 4% opacity
- No gradients that feel colorful or dramatic
- No images, no textures, no blurred photos

**Why this background:**

- Pure dark (#0b0f19) with zero texture feels like "just a black page" — the pattern adds depth without noise
- The grid/dot pattern subtly signals "structured systems" — reinforcing Appriyo's identity
- Blue dots at 4% opacity barely visible but subconsciously connect to the brand color

**Background implementation option:**

```css
.hero-background {
  background-color: #0b0f19;
  background-image: radial-gradient(circle, #2563eb 1px, transparent 1px);
  background-size: 32px 32px;
  background-position: center;
  opacity: 1; /* Pattern opacity controlled by the rgba above */
}
```

**Alternatives rejected:**

| Option                                   | Rejection Reason                            |
| ---------------------------------------- | ------------------------------------------- |
| Full gradient (dark blue to dark purple) | Too dramatic — feels like a SaaS startup ad |
| Animated particle background             | Performance hit + screams "generic tech"    |
| Hero image (city, office, laptop)        | Stock photography — kills trust instantly   |
| Pure solid dark, zero pattern            | Feels unfinished and flat                   |
| Glassmorphism elements                   | Trendy, not timeless — violates design.md   |

# LAYER 5 — EMOTIONAL DIRECTION

---

## 5.1 How the Visitor Should Feel

When a business owner lands on the Appriyo hero, they should feel:

| Feeling        | How the design creates it                                                  |
| -------------- | -------------------------------------------------------------------------- |
| **Recognized** | The headline names their exact situation — "still manual?"                 |
| **Calm**       | Dark, low-noise background. No animations screaming at them.               |
| **Clear**      | One message. One action. Nothing competing for attention.                  |
| **Trusting**   | Simple language. No hype. Specific tools named (paper, WhatsApp).          |
| **Competent**  | "No tech knowledge required" — they don't need to know anything to proceed |

---

## 5.2 Five-Word Vibe

```
CLEAR. CALM. DIRECT. TRUSTWORTHY. PRACTICAL.
```

This is the emotional fingerprint of the hero.
Use these five words as a filter for every design decision.

If a proposed element feels:

- clever → remove it (not "clear")
- exciting → tone it down (not "calm")
- vague → rewrite it (not "direct")
- boastful → remove it (not "trustworthy")
- theoretical → make it concrete (not "practical")

---

## 5.3 "This Should NOT Feel Like..."

| Should NOT feel like                   | Why this matters                                             |
| -------------------------------------- | ------------------------------------------------------------ |
| A VC-funded startup trying to raise    | Appriyo's target doesn't relate to that world                |
| A generic IT outsourcing company       | Every IT company has the same hero — Appriyo must stand out  |
| A SaaS product with a free trial       | Appriyo sells consultation, not a commodity                  |
| A design agency                        | We're not selling aesthetics                                 |
| A corporate enterprise software vendor | Too cold, too large, too complex                             |
| A trendy app with a flashy launch      | Our visitors are not trend-followers                         |
| Overwhelming                           | Any hero where you don't know where to look first has failed |

---

## 5.4 Reference Points (Brand Benchmarks)

**Appriyo's hero should feel closer to:**

- A quiet, confident conversation
- A direct invoice — clear, structured, no decoration
- A trusted local professional who knows their craft

**Not like:**

- A product launch event
- A Dribbble design exercise
- A Silicon Valley pitch deck

# LAYER 6 — INTERACTION & BEHAVIOR

---

## 6.1 Load Behavior

The hero content appears with a simple fade-up animation on page load.

**Animation spec:**

```js
// Framer Motion — hero content entrance
const heroEntrance = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.5, ease: "easeOut" },
};

// Staggered entrance for elements
const stagger = {
  initial: {},
  animate: { transition: { staggerChildren: 0.1 } },
};
```

**Stagger order:**

1. Pre-heading badge — delay 0
2. H1 headline — delay 0.1s
3. Subheadline — delay 0.2s
4. Supporting line — delay 0.3s
5. CTA row — delay 0.4s
6. Micro trust line — delay 0.5s

**Total entrance duration:** ~0.9s for all elements to settle

---

## 6.2 Background Pattern Behavior

The background dot/grid pattern is static — it does NOT move, parallax, or animate.

**Rule:** Static background only. Any movement behind text creates reading friction.

---

## 6.3 CTA Hover Behavior

| CTA                 | Hover State                                                   | Transition |
| ------------------- | ------------------------------------------------------------- | ---------- |
| Primary (filled)    | Background darkens to `--color-primary-dark` (#1e40af)        | 200ms ease |
| Secondary (outline) | Background fills to `--color-surface` (#111827), border stays | 200ms ease |

**Scale on hover:** Neither CTA scales. Scale effects on buttons feel gamey, not professional.

---

## 6.4 Scroll Indicator

A minimal animated scroll indicator at the bottom of the hero.

**Spec:**

- A small chevron-down icon (Lucide `ChevronDown`)
- Color: `--color-text-muted` (#6b7280)
- Animation: slow vertical bob — translateY 0 → 6px → 0, loop, 1.5s duration, ease-in-out
- Clicking it scrolls to `#services`
- Disappears once the user scrolls past the hero

**Rule:** This is the ONLY looping animation on the site. It's acceptable because it's a navigation signal, not decoration.

---

## 6.5 Reduced Motion

All entrance animations are disabled when `prefers-reduced-motion: reduce` is detected.

```js
import { useReducedMotion } from "framer-motion";

function HeroSection() {
  const shouldReduce = useReducedMotion();

  const animation = shouldReduce
    ? { initial: false } // No animation
    : { initial: { opacity: 0, y: 20 }, animate: { opacity: 1, y: 0 } };

  return <motion.div {...animation}>...</motion.div>;
}
```

| Rule                            | Value                   | Reason                                  |
| ------------------------------- | ----------------------- | --------------------------------------- |
| No video background             | Ever                    | ~5MB+ cost, major CLS impact            |
| No JavaScript parallax          | Ever                    | Janky scroll, kills Lighthouse score    |
| Background pattern via CSS only | No image file           | Zero network cost                       |
| Font preloaded                  | Inter from Google Fonts | Prevents FOUT                           |
| Hero is above-the-fold          | No lazy loading here    | This must render immediately            |
| Framer Motion tree-shaken       | Import only what's used | Bundle size                             |
| Scroll indicator is CSS         | `@keyframes` bob        | Not Framer Motion — it's always looping |

**Hero target metrics:**

- Hero section renders: < 0.5s after page load
- No Cumulative Layout Shift from hero: CLS = 0 for this section
- LCP element (H1) loads: < 1.5s

## 7.4 CTA Behavior on Mobile

- Desktop: Side-by-side (flex row), gap-4
- Tablet: Side-by-side if they fit, otherwise stacked
- Mobile: Stacked (flex column), both full-width
- Primary CTA always appears FIRST (above secondary)
- Minimum touch target: 48px height on mobile

---

## 7.5 Hero Height by Breakpoint

| Breakpoint | Hero Height                                                                                 |
| ---------- | ------------------------------------------------------------------------------------------- |
| Desktop    | `100vh` (full viewport)                                                                     |
| Tablet     | `90vh`                                                                                      |
| Mobile     | `auto` with `min-height: 100svh` (uses small viewport height to account for browser chrome) |

**Why `min-height: 100svh` on mobile:** On iOS Safari, `100vh` includes the URL bar space, causing the hero to be taller than the visible screen. `100svh` (small viewport height) uses the actual visible height.

---

## 7.6 Background Pattern on Mobile

The dot grid pattern scale reduces on mobile:

```css
/* Desktop */
background-size: 32px 32px;

/* Mobile */
background-size: 24px 24px;
```

This keeps the pattern proportional and prevents it from looking too sparse on smaller screens.

# LAYER 8 — REJECTION CRITERIA

---

## 8.1 Reject the Hero Design If...

The following are HARD rejections. If any condition is true, the hero must be redesigned or rewritten before it goes live.

```
MESSAGING REJECTIONS
[ ] The headline is longer than 8 words → Rewrite
[ ] The headline contains any of: "innovative", "cutting-edge", "solutions",
    "transform", "leverage", "empower" → Rewrite
[ ] The headline is company-first ("Appriyo is...", "Welcome to...") → Rewrite
[ ] A 45-year-old non-technical business owner cannot understand the headline in 3 seconds → Rewrite
[ ] The CTA says "Get Started", "Submit", "Click Here", or "Learn More" → Rewrite
[ ] The subheadline doesn't name a specific pain point → Rewrite
[ ] The hero could belong to any generic IT company → Rewrite

DESIGN REJECTIONS
[ ] A stock photo of people, offices, or laptops is used → Remove
[ ] A gradient blob or abstract 3D shape is used → Remove
[ ] An animated particle background is used → Remove
[ ] More than 2 colors are active simultaneously → Simplify
[ ] The accent/violet color appears in the hero → Move it below
[ ] Text contrast ratio below 4.5:1 → Fix colors
[ ] CTA is not the most visually prominent interactive element → Fix hierarchy

BEHAVIOR REJECTIONS
[ ] Any animation takes longer than 0.5s → Shorten it
[ ] Any element moves while the user reads (parallax, float) → Remove it
[ ] The page has Cumulative Layout Shift > 0.1 from hero elements → Fix loading order
[ ] CTA buttons are smaller than 44px tall on mobile → Fix touch targets

EXPERIENCE REJECTIONS
[ ] The hero takes longer than 1s to be readable → Optimize
[ ] The visitor cannot see the CTA without scrolling on mobile → Fix
[ ] The message changes meaning on mobile (truncated headline) → Fix text
[ ] The hero looks significantly different on iOS Safari → Fix
```
