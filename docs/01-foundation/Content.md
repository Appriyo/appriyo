# 1. THE CONTENT PHILOSOPHY

## 1.1 One Rule Above All Others

> **Write for the business owner who is skeptical, busy, and non-technical.**
> **Not for other developers. Not for investors. Not for yourself.**

The person reading this website runs a small business. They've probably been burned by tech promises before. They don't have time to decode jargon. They need to know in 10 seconds whether Appriyo can help them.

Every word is written for that person.

---

## 1.2 The Three Content Tests

Before any copy goes live, it must pass all three:

**Test 1 — The Business Owner Test**
> Could a 45-year-old repair shop owner in Khulna read this and understand it immediately?
> If NO → rewrite it.

**Test 2 — The So What Test**
> Does this sentence communicate a benefit, a problem, or an action?
> If it's just describing Appriyo — delete it.

**Test 3 — The Competitor Test**
> Could this exact sentence appear on any other IT company's website?
> If YES → it's too generic. Make it specific.

# 2. TONE OF VOICE

## 2.1 What Appriyo Sounds Like

| Attribute      | What it means in practice                                   |
|----------------|-------------------------------------------------------------|
| **Direct**     | Say the thing. No preamble. No "We are pleased to offer..." |
| **Calm**       | No exclamation points. No hype. Confidence, not excitement. |
| **Honest**     | Never overpromise. Never claim what we can't deliver.       |
| **Empathetic** | Acknowledge the problem before offering the solution.       |
| **Simple**     | If a simpler word exists — use it. Always.                  |
| **Grounded**   | No abstract language. Talk about real things.               |

## 2.2 The Tone Spectrum

Appriyo sits at a specific point on each spectrum:

```
Formal ←————————[Appriyo]——————→ Casual
Corporate ←——[Appriyo]—————————→ Conversational
Technical ←—————————[Appriyo]——→ Plain
Cold ←——————————[Appriyo]———————→ Warm
Humble ←————[Appriyo]———————————→ Confident
```

Appriyo is: **lightly conversational, plain, warm, and quietly confident.**

# 3. WRITING STYLE RULES

## 3.1 Sentence Rules

| Rule                          | Detail                                                     |
|-------------------------------|------------------------------------------------------------|
| **One idea per sentence**     | Split compound sentences. Always.                          |
| **Short sentences preferred** | Average 12 words. Max 20 words per sentence.               |
| **Active voice always**       | "We build the system" not "The system is built by us"      |
| **Present tense**             | "Amar Repair tracks every job" not "will track"            |
| **No passive hedging**        | Not "may be able to help" — either "we help" or "we don't" |
| **No filler openers**         | Never start with "At Appriyo, we believe..."               |
| **Second person**             | Use "you" and "your" — not "clients" or "businesses"       |

## 3.2 Paragraph Rules

| Rule                                   | Detail                                                              |
|----------------------------------------|---------------------------------------------------------------------|
| Max 3 lines per paragraph on desktop   | After that — break it                                               |
| Max 2 paragraphs before a visual break | Use lists, cards, or whitespace                                     |
| No padding paragraphs                  | Every paragraph must add information — not restate what came before |
| No throat-clearing                     | Don't explain what you're about to say — just say it                |

## 3.3 Punctuation Rules

| Rule                           | Detail                                        |
|--------------------------------|-----------------------------------------------|
| **No exclamation points**      | They signal hype. We don't hype.              |
| **Em dash allowed**            | Used sparingly for emphasis — like this       |
| **No ellipses**                | Creates vagueness. Say the complete thought.  |
| **Oxford comma**               | Always: "automation, custom software, and AI" |
| **No all-caps for emphasis**   | Use bold or sentence structure instead        |
| **No multiple question marks** | One is enough. Ever.                          |

# 4. HEADLINE RULES

Headlines are the most important copy on the page. They are read first and must work alone.

## 4.1 Headline Principles

| Principle                               | Detail                                                                             |
|-----------------------------------------|------------------------------------------------------------------------------------|
| **Problem-first or Outcome-first**      | Never "About Appriyo" — always "Why Your Business Needs a System, Not More Staff"  |
| **Specific over vague**                 | "Repair shop software that tracks every job" beats "Powerful business software"    |
| **No questions that feel rhetorical**   | "Are you ready to transform your business?" → No one answers yes to this           |
| **No word-for-word company name in H1** | The page context and logo say "Appriyo" — the headline should say something useful |
| **Max 10 words for H1**                 | Longer → split into headline + subheadline                                         |
| **Subheadline does the explaining**     | Headline = hook. Subheadline = the full thought.                                   |

## 4.2 Headline Formulas

These are proven structures — use them:

**Formula 1: The Problem Mirror**
> "Still [doing the painful thing they're doing]?"
> Example: "Still tracking repair jobs on paper?"

**Formula 2: The Outcome Promise**
> "[Result] — without [the thing they fear]."
> Example: "A complete shop system — without the complexity."

**Formula 3: The Direct Statement**
> "[What it does]. [For whom]."
> Example: "Batch management for educators who are done with notebooks."

**Formula 4: The Before/After**
> "From [painful current state] to [desired outcome]."
> Example: "From WhatsApp threads to a real order system."

**Formula 5: The Simple Claim**
> "[Short, specific, true claim about what Appriyo does]."
> Example: "Four services. Focused on one goal."

## 4.3 Headline Anti-Patterns

| ❌ Anti-pattern                        | Why it fails                     |
|---------------------------------------|----------------------------------|
| "Welcome to Appriyo"                  | Tells the visitor nothing useful |
| "Your Digital Transformation Partner" | Vague, used by everyone          |
| "Innovation. Simplicity. Excellence." | Three words that say nothing     |
| "We Help Businesses Grow"             | Every company claims this        |
| "Next-Generation Business Solutions"  | Meaningless buzzwords            |
| "Building a Better Digital Future"    | Abstract, unverifiable           |

# 5. CTA STYLE RULES

CTAs are the conversion mechanism. Every word matters.

## 5.1 CTA Principles

| Principle                        | Detail                                                             |
|----------------------------------|--------------------------------------------------------------------|
| **Specific and action-oriented** | The button should describe what happens next, not just urge action |
| **First-person where possible**  | "Start My Free Consultation" beats "Get a Free Consultation"       |
| **No "Click Here"**              | Never. It describes the mechanic, not the outcome.                 |
| **No "Learn More" alone**        | Add context: "Learn More About Amar Repair"                        |
| **No "Submit"**                  | Replace with the actual action: "Send My Message"                  |
| **Low-commitment language**      | "Let's Talk" beats "Buy Now" for a consulting business             |
| **Consistent across the site**   | Primary CTA toward contact must feel the same everywhere           |

## 5.2 CTA Copy by Location

| Location            | CTA Text                         | Destination         |
|---------------------|----------------------------------|---------------------|
| Hero (primary)      | "Let's Talk About Your Business" | `/contact`          |
| Hero (secondary)    | "See What We Build"              | `#products`         |
| Services section    | "See Full Service Details"       | `/services`         |
| Solutions card      | "See [Product Name]"             | Product page        |
| Product card        | "Learn More About [Product]"     | Product detail page |
| Why Appriyo         | "See How We Work"                | `#approach`         |
| About preview       | "Meet the Full Team"             | `/about`            |
| Every page end CTA  | "Get a Free Consultation"        | `/contact`          |
| Navbar              | "Contact Us"                     | `/contact`          |
| Footer              | "Book a Consultation"            | `/contact`          |
| Product page        | "Talk to Us About [Product]"     | `/contact`          |
| Solutions page end  | "Tell Us Your Problem"           | `/contact`          |
| Contact form button | "Send Message"                   | Form submit         |

## 5.3 CTA Anti-Patterns

| ❌ Anti-pattern       | Why it fails                                                |
|----------------------|-------------------------------------------------------------|
| "Click Here"         | Describes the mechanic, not the outcome                     |
| "Submit"             | Cold, bureaucratic, form-centric                            |
| "Get Started!"       | Started with what? Vague + exclamation point                |
| "Learn More" (alone) | Where? About what?                                          |
| "Contact Us Today!"  | Exclamation point + "today" sounds pushy                    |
| "Request a Quote"    | Feels transactional — we sell consultation, not commodities |

# 6. MESSAGING PRINCIPLES

## 6.1 The Core Message Architecture

Every page and section supports one of four core messages. All content must map to one:

| Message                          | Used In                         | Key Words                                           |
|----------------------------------|---------------------------------|-----------------------------------------------------|
| **"We understand your problem"** | Hero, Solutions, Product heroes | manual, paper, WhatsApp, track, lose, forget        |
| **"We've solved it before"**     | Products, Why Appriyo           | Amar Repair, Amar Batch, built for, real businesses |
| **"We're trustworthy people"**   | About, Approach, Team           | small team, direct, stay, fix, long-term            |
| **"The next step is easy"**      | Contact, CTAs, Form             | conversation, no pressure, 1 business day, simple   |

## 6.2 Messaging Hierarchy per Section

| Section     | Lead Message                 | Supporting Message           |
|-------------|------------------------------|------------------------------|
| Hero        | "We understand your problem" | "We've solved it before"     |
| Services    | "We've solved it before"     | "We're trustworthy people"   |
| Solutions   | "We understand your problem" | "We've solved it before"     |
| Products    | "We've solved it before"     | "The next step is easy"      |
| Why Appriyo | "We're trustworthy people"   | "We understand your problem" |
| Approach    | "We're trustworthy people"   | "The next step is easy"      |
| About       | "We're trustworthy people"   | "We've solved it before"     |
| Contact     | "The next step is easy"      | "We're trustworthy people"   |

## 6.3 The Problem-First Rule

Every product page, solution card, and service block must begin by stating the customer's problem — not Appriyo's solution.

```
WRONG ORDER:
"Amar Repair is a comprehensive repair shop management system
that helps shop owners track jobs, customers, and payments."

CORRECT ORDER:
"Managing repair jobs on paper or WhatsApp means lost orders,
missed customers, and hours of avoidable confusion.
Amar Repair gives your shop one system for everything."
```

The customer must recognize themselves before they care about what you offer.

# 8. NUMBERS AND SPECIFICITY RULES

Specific numbers build trust. Vague claims destroy it.

| Rule                                 | Detail                                                      |
|--------------------------------------|-------------------------------------------------------------|
| **Use real numbers where available** | "4 core services" not "a range of services"                 |
| **Don't invent statistics**          | Never say "90% of businesses..." without a source           |
| **Use time specifics**               | "We respond within 1 business day" not "quickly"            |
| **Name the team size**               | "A team of four" — not "a small, agile team"                |
| **Name business hours exactly**      | "Sunday–Thursday, 9 AM–6 PM (GMT+6)" — not "business hours" |
| **No "many" or "some"**              | Say the actual number or remove the claim                   |
| **No "years of experience"**         | Unless it's real and verified — then state it exactly       |

# 9. FORMATTING RULES FOR WEB COPY

## 9.1 When to Use Lists

| Use lists for          | Don't use lists for                       |
|------------------------|-------------------------------------------|
| Features (3+ items)    | Philosophy or values (prose reads better) |
| Step-by-step processes | Single-point statements                   |
| What you do / don't do | Continuous narrative content              |
| Examples (4+ items)    | Anything under 3 items                    |

## 9.2 Bold and Emphasis

| Use bold for                          | Don't use bold for         |
|---------------------------------------|----------------------------|
| Key product names inline              | Decorative emphasis        |
| Critical warnings or constraints      | General interest phrases   |
| Numbers that matter ("5 fields only") | Random words in a sentence |

**Rule:** If everything is bold, nothing is.

## 9.3 Link Text Rules

| ❌ Don't write              | ✅ Write instead                 |
|----------------------------|---------------------------------|
| "Click here to learn more" | "See how Amar Repair works"     |
| "Read more"                | "Read the full service details" |
| "Here" (as a link)         | The destination or action       |

# 10. CONTENT SPECIFIC TO KEY SECTIONS

## 10.1 Error and Edge-Case Messages

These are often forgotten — they matter for trust.

| State                | Message                                                                                                                      |
|----------------------|------------------------------------------------------------------------------------------------------------------------------|
| Form success         | "We've received your message. Someone from the Appriyo team will reach out within 1 business day via your preferred method." |
| Form error           | "Something went wrong. Please email us at contact@appriyo.com or WhatsApp us at +880 1410394038."                            |
| 404 page             | "This page doesn't exist. You may have followed an old link." + link to homepage                                             |
| Empty state (future) | Describe clearly what goes here — never "No data found"                                                                      |
| Loading state        | "Sending your message..."                                                                                                    |

## 10.2 Microcopy Rules

Microcopy = labels, placeholders, helper text, button states.

| Element                  | Rule                                                                                                 |
|--------------------------|------------------------------------------------------------------------------------------------------|
| Form placeholders        | Describe what to write, not just "Enter here": `"Your name"`, `"What's slowing your business down?"` |
| Form labels              | Visible always — never placeholder-only forms                                                        |
| Required field indicator | Use "(required)" in label text — not just a red asterisk                                             |
| Button loading state     | `"Sending..."` while submitting — not a spinner alone                                                |
| Helper text              | Appears below field when relevant: `"We'll use this to contact you first"`                           |

# 11. CONTENT QUALITY CHECKLIST

Before any copy is finalized, run through this checklist:

```
CLARITY
[ ] Can a non-technical business owner understand this in 5 seconds?
[ ] Is there any jargon that should be replaced with plain language?
[ ] Does the headline tell the visitor what to expect from this section?

SPECIFICITY
[ ] Are all claims specific and verifiable?
[ ] Has "many", "some", "various", or "numerous" been removed?
[ ] Are there concrete examples where claims are made?

TONE
[ ] Is every sentence in active voice?
[ ] Have all exclamation points been removed?
[ ] Does this sound like Appriyo — calm, direct, honest?

STRUCTURE
[ ] Is each paragraph under 3 lines?
[ ] Does every section end with a clear next step?
[ ] Is the problem stated before the solution on every product/solution page?

CTAs
[ ] Does every CTA describe what happens next?
[ ] Has "Submit", "Click Here", or "Learn More" (alone) been removed?
[ ] Is the CTA text consistent with the CTA style guide in §5.2?
```

# 12. LOCALIZATION NOTE

The Appriyo website launches in English.

The primary market is small businesses in Bangladesh — many of whom are comfortable reading English in a professional context. Bangla version is a future consideration.

**Rules for English copy targeting a Bangladeshi audience:**
- Use standard international English — not American or British slang
- Avoid idioms that may not translate: "hit the ground running", "ballpark figure"
- Currency references: Use BDT (৳) in product pricing contexts — not just "$"
- Time references: GMT+6 always, spelled out fully
- Phone format: +880 prefix always included
